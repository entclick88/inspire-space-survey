# วิธีติดตั้งระบบแบบสำรวจ Inspire Space (โฮสต์หน้าเว็บบน GitHub Pages)

ระบบแบ่งเป็น 2 ส่วน:

- **Backend** — `Code.gs` รันบน Google Apps Script ทำหน้าที่บันทึกคำตอบลง Google Sheet และกันการส่งซ้ำ
- **Frontend** — `index.html` โฮสต์บน GitHub Pages เรียก backend ผ่าน HTTP

## ขั้นที่ 1: ติดตั้ง Backend (Apps Script)

1. เปิด Google Sheet ปลายทาง
   https://docs.google.com/spreadsheets/d/1pol6seeM23mKN0MU2KuO9dvO-ZagkTPm2icyqJ5DaL8/edit
2. เมนู **ส่วนขยาย (Extensions) → Apps Script**
3. ลบโค้ดเดิมในไฟล์ `Code.gs` แล้ววางเนื้อหาจากไฟล์ `Code.gs` ของโปรเจกต์นี้
4. (ทางเลือก) ถ้าอยากให้เปิดฟอร์มจากลิงก์ Apps Script ตรงๆ ได้ด้วย ให้เพิ่มไฟล์ HTML ชื่อ `index` แล้ววางเนื้อหา `index.html` — ไม่จำเป็นถ้าใช้ GitHub Pages อย่างเดียว
5. กด **Deploy → New deployment**
   - ประเภท: **Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
   - กด **Deploy** และอนุญาตสิทธิ์
6. คัดลอก **Web app URL** (รูปแบบ `https://script.google.com/macros/s/.../exec`)

## ขั้นที่ 2: ใส่ URL ลงใน index.html

เปิด `index.html` หาบรรทัด:

```js
var SCRIPT_URL = 'https://script.google.com/macros/s/ใส่_DEPLOYMENT_ID_ที่นี่/exec';
```

แทนที่ด้วย Web app URL ที่ได้จากขั้นที่ 1 แล้ว commit + push ขึ้น GitHub

## ขั้นที่ 3: เปิด GitHub Pages

1. ที่ repo บน GitHub ไปที่ **Settings → Pages**
2. Source: **Deploy from a branch** → Branch: `main` / โฟลเดอร์ `/ (root)` → Save
3. รอสักครู่ จะได้ลิงก์ `https://<username>.github.io/<repo>/` นำไปแจกนักเรียนได้เลย

> หมายเหตุ: ทุกครั้งที่แก้โค้ดใน Apps Script ต้อง Deploy ใหม่ (Manage deployments → Edit → New version) URL เดิมจึงจะได้โค้ดใหม่

## การทำงานของระบบกันส่งซ้ำ (1 เครื่อง ตอบได้ 1 ครั้ง)

1. **Device ID ใน localStorage** — เครื่องที่เคยส่งแล้วจะถูกจดจำในเบราว์เซอร์ เปิดลิงก์อีกครั้งจะเห็นหน้า "เครื่องนี้ได้ส่งคำตอบแล้ว" ทันที
2. **ตรวจสอบซ้ำฝั่งเซิร์ฟเวอร์** — ทุกครั้งที่ส่ง ระบบเช็ค Device ID และ Fingerprint ของเครื่องกับข้อมูลใน Sheet ถ้าซ้ำจะไม่บันทึก

> ข้อจำกัด: หากนักเรียนล้างข้อมูลเบราว์เซอร์และเปลี่ยนเบราว์เซอร์/อุปกรณ์ อาจส่งซ้ำได้ วิธีนี้เป็นการป้องกันระดับทั่วไปที่ไม่ต้องให้นักเรียนล็อกอิน หากต้องการกันซ้ำแบบเด็ดขาด ต้องบังคับล็อกอินด้วยบัญชีโรงเรียนแทน

## โครงสร้างข้อมูลใน Sheet

แถวหัวคอลัมน์จะถูกสร้างอัตโนมัติเมื่อมีการส่งครั้งแรก:

| คอลัมน์ | ข้อมูล |
|---|---|
| A | Timestamp (เวลาที่ส่ง) |
| B | Device ID |
| C | Fingerprint |
| D–E | เพศ, ระดับชั้น |
| F–Q | คำตอบ 12 ข้อ (เห็นด้วย / ไม่เห็นด้วย / ไม่แสดงความคิดเห็น) |
